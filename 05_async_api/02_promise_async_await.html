02_Promise와 async/await

이 파일에서는 콜백 중첩 문제를 줄이기 위해 사용하는 Promise와, Promise 코드를 더 읽기 쉽게 작성하는 async/await 문법을 확인한다.

1. Promise 생성과 상태

Promise는 비동기 작업의 성공 또는 실패 결과를 표현하는 객체이다. 작업이 끝나기 전에는 pending, 성공하면 fulfilled, 실패하면 rejected 상태가 된다.

Promise 안에서는 성공 결과를 resolve()로 전달하고, 실패 결과를 reject()로 전달한다. 

버튼을 클릭하면 Promise 처리 결과가 표시된다.

2. then/catch 체이닝

then()은 Promise가 성공했을 때 실행할 코드를 작성한다. catch()는 중간에 실패가 발생했을 때 실행할 코드를 작성한다.

then() 안에서 다시 Promise를 반환하면 다음 then()으로 결과를 이어갈 수 있다. 이를 Promise 체이닝이라고 한다. 

버튼을 클릭하면 결과가 순서대로 표시된다.

3. async/await

async/await는 Promise 기반 비동기 코드를 더 읽기 쉽게 작성하기 위한 문법이다. async가 붙은 함수 안에서 await를 사용할 수 있다.

await는 Promise가 처리될 때까지 기다린 뒤 성공 결과를 반환한다. 그래서 비동기 코드를 위에서 아래로 실행되는 코드처럼 읽을 수 있다. 

버튼을 클릭하면 결과가 순서대로 표시된다.

4. then 방식과 async/await 방식 비교

두 방식 모두 Promise를 처리하는 문법이다. then()은 Promise 흐름을 메서드 체이닝으로 직접 연결하는 방식이고, async/await는 같은 흐름을 순서대로 읽히도록 작성하는 방식이다.

수업에서는 두 방식을 모두 확인하되, 이후 API 통신 예제에서는 실무 코드에서 자주 사용하는 async/await 중심으로 작성한다.
