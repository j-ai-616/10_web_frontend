# 00. 비동기 처리와 API 통신 기본 개요

## 1. 이 단원에서 다루는 내용

브라우저에서 시간이 걸리는 작업을 어떻게 처리하는지 학습한다.
대표적인 예는 서버에 데이터를 요청하는 API 통신이다.

1. 동기 처리, 비동기 처리, 콜백 함수
2. Promise, then/catch, async/await
3. fetch를 활용한 API 요청
4. axios를 활용한 API 요청과 fetch와의 차이

## 2. 동기 처리와 비동기 처리

동기 처리는 앞 작업이 끝나야 다음 작업이 실행되는 방식이다.
일반적인 JavaScript 코드는 위에서 아래로 순서대로 실행되므로 기본적으로 동기적으로 읽힌다.

비동기 처리는 시간이 걸리는 작업을 기다리는 동안 다음 코드를 먼저 실행할 수 있는 방식이다.
서버 요청, 타이머, 사용자 이벤트처럼 완료 시점을 바로 알 수 없는 작업에서 자주 사용한다.

브라우저에서 서버 응답을 기다리는 동안 화면 전체가 멈춘다면 사용자는 불편함을 느낀다.
그래서 JavaScript는 시간이 걸리는 작업을 비동기로 처리하고, 작업이 끝난 뒤 실행할 코드를 따로 연결할 수 있도록 한다.

## 3. 콜백 함수

콜백 함수는 다른 함수에 인자로 전달되어 나중에 실행되는 함수이다.
비동기 작업에서는 작업이 끝난 뒤 실행할 코드를 콜백으로 전달할 수 있다.

콜백은 간단한 흐름에서는 직관적이지만, 비동기 작업을 여러 단계로 순서대로 연결하면 함수가 계속 안쪽으로 중첩된다.
이처럼 코드가 오른쪽으로 깊게 들어가 읽기 어려워지는 문제를 콜백 중첩 문제 또는 콜백 지옥이라고 부른다.

## 4. Promise

Promise는 비동기 작업의 성공 또는 실패 결과를 표현하는 객체이다.
콜백을 직접 중첩하는 대신, 비동기 작업의 결과를 객체로 반환하고 그 결과를 `then()`, `catch()`로 이어서 처리할 수 있게 해준다.

Promise는 다음 세 가지 상태를 가진다.

1. pending
   - 비동기 작업이 아직 끝나지 않은 상태이다.

2. fulfilled
   - 비동기 작업이 성공한 상태이다.
   - 이때 `resolve()`로 전달한 값이 성공 결과가 된다.

3. rejected
   - 비동기 작업이 실패한 상태이다.
   - 이때 `reject()`로 전달한 값이 실패 원인이 된다.

## 5. then/catch

`then()`은 Promise가 성공했을 때 실행할 코드를 작성하는 메서드이다.
`catch()`는 Promise 처리 중 실패가 발생했을 때 실행할 코드를 작성하는 메서드이다.

`then()` 안에서 또 다른 Promise를 반환하면 다음 `then()`으로 결과를 이어갈 수 있다.
이를 Promise 체이닝이라고 한다.

## 6. async/await

`async/await`는 Promise를 더 읽기 쉬운 형태로 작성하기 위한 문법이다.

`async`는 함수 앞에 작성한다.
`async`가 붙은 함수는 항상 Promise를 반환한다.

`await`는 Promise를 반환하는 작업 앞에 작성한다.
`await`를 만나면 해당 Promise가 처리될 때까지 기다린 뒤, 성공 결과를 변수에 담아 다음 줄을 실행한다.

```javascript
async function callAPI() {
    const response = await fetch('https://jsonplaceholder.typicode.com/users');
    const users = await response.json();
    console.log(users);
}
```

`await`를 사용하면 비동기 코드를 동기 코드처럼 위에서 아래로 읽을 수 있다.
다만 `await`는 `async` 함수 안에서 사용해야 하며, 실패 처리는 일반적으로 `try/catch`를 사용한다.

## 7. fetch

`fetch()`는 브라우저에서 기본으로 제공하는 HTTP 요청 함수이다.
별도 라이브러리를 추가하지 않아도 사용할 수 있으며, 요청 결과를 Promise로 반환한다.

`fetch()`를 사용할 때는 응답 본문을 바로 사용할 수 없다.
응답 객체에서 `response.json()`을 호출해야 JSON 문자열이 JavaScript 객체로 변환된다.

또한 `fetch()`는 HTTP 상태 코드가 404나 500이어도 네트워크 요청 자체가 완료되면 바로 `catch()`로 이동하지 않는다.
따라서 `response.ok`를 확인해서 직접 에러를 발생시키는 처리가 필요하다.

## 8. axios

axios는 HTTP 요청을 쉽게 처리하기 위한 외부 라이브러리이다.
브라우저 기본 기능은 아니므로 CDN으로 불러오거나 npm으로 설치해야 한다.

axios는 응답 본문을 자동으로 JavaScript 객체로 변환하고, 변환된 데이터를 `response.data`에 담아준다.
또한 요청 파라미터, 기본 URL, 공통 헤더, 인터셉터 같은 설정을 관리하기 편해 실무 프로젝트에서도 자주 사용된다.
