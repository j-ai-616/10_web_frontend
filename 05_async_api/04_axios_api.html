04_axios API 요청

axios는 HTTP 요청을 쉽게 처리하기 위한 외부 라이브러리이다. 브라우저 기본 기능은 아니므로 CDN이나 npm을 통해 추가해야 한다.

이 파일에서는 axios의 기본 GET 요청, params 전달, POST 요청, 공통 설정을 확인하고 fetch와 어떤 점이 다른지 비교한다.

1. axios 기본 GET 요청

axios는 응답 본문을 자동으로 JavaScript 객체로 변환해준다. 따라서 fetch()처럼 직접 response.json()을 호출하지 않아도 된다.

axios 응답 객체에는 여러 정보가 들어 있으며, 실제 응답 본문은 response.data에 들어 있다.

2. axios params와 fetch URLSearchParams 비교

5. fetch와 axios의 차이 정리

둘 다 API 요청에 사용할 수 있지만 기본 사용 방식에 차이가 있다.

  • fetch()는 브라우저 기본 기능이므로 별도 라이브러리가 필요 없다.
  • axios는 외부 라이브러리이므로 CDN 또는 패키지 설치가 필요하다.
  • fetch()는 응답 본문을 직접 response.json()으로 변환해야 한다.
  • axios는 응답 본문을 response.data로 바로 사용할 수 있다.
  • fetch()는 HTTP 오류 상태를 직접 response.ok로 확인하는 코드가 필요하다.
  • axios는 400번대, 500번대 응답을 기본적으로 에러로 처리해 catch에서 다루기 쉽다.
  • axios는 params, baseURL, timeout 같은 요청 설정을 객체 형태로 관리하기 편하다.
