# 00. JavaScript 기본 개요

## 1. JavaScript란?

JavaScript는 웹 브라우저에서 동작하는 프로그래밍 언어이다.

HTML이 화면의 구조를 만들고, CSS가 화면의 디자인을 담당한다면, JavaScript는 화면의 동작을 담당한다.

예를 들어 JavaScript로 다음과 같은 기능을 만들 수 있다.

1. 버튼을 클릭했을 때 화면 내용 변경
2. 입력값 검사
3. 목록 추가와 삭제
4. 이미지 또는 스타일 변경
5. 서버에서 받은 데이터를 화면에 표시

이번 과정에서는 JavaScript를 깊은 알고리즘 언어로 다루기보다, 웹 화면을 동적으로 제어하기 위한 기본 문법과 DOM 조작 중심으로 학습한다.

## 2. JavaScript와 ECMAScript

JavaScript의 핵심 문법 표준을 ECMAScript라고 한다.

ECMAScript는 변수, 자료형, 연산자, 함수, 객체, 배열 같은 언어의 기본 규칙을 정의한다.

브라우저에서 사용하는 JavaScript는 ECMAScript에 더해 브라우저가 제공하는 기능을 함께 사용한다.  
대표적인 브라우저 기능은 다음과 같다.

1. DOM
   - HTML 요소를 선택하고 변경하는 기능이다.

2. BOM
   - 브라우저 창, 주소, 기록 등 브라우저 환경을 다루는 기능이다.

3. 이벤트
   - 클릭, 입력, 제출 같은 사용자 동작을 처리하는 기능이다.

## 3. 브라우저 환경과 Node.js 환경

JavaScript는 브라우저에서도 실행할 수 있고 Node.js에서도 실행할 수 있다.

두 환경 모두 JavaScript 기본 문법은 실행할 수 있다.  
하지만 제공하는 기능은 다르다.

1. 브라우저 환경
   - HTML, CSS와 함께 동작한다.
   - `document`, `window` 같은 객체를 사용할 수 있다.
   - DOM을 통해 화면 요소를 선택하고 변경할 수 있다.

2. Node.js 환경
   - 브라우저 밖에서 JavaScript를 실행하는 환경이다.
   - 파일 시스템, 서버 개발, 패키지 관리 등에 사용된다.
   - 기본적으로 DOM API를 제공하지 않는다.

이번 수업의 목표는 웹 화면 제어이므로 브라우저 환경을 기준으로 진행한다.  
실행 결과는 브라우저 개발자 도구의 Console 탭에서 확인한다.

## 4. JavaScript 실행 방법

HTML 문서 안에서 JavaScript를 실행하는 방법은 크게 두 가지이다.

## 4-1. HTML 내부에 작성

```html
<script>
    console.log('JavaScript 실행');
</script>
```

간단한 예제 확인에 사용할 수 있다.

## 4-2. 외부 JavaScript 파일 연결

```html
<script src="app.js"></script>
```

JavaScript 코드가 많아지면 별도의 `.js` 파일로 분리하는 것이 좋다.

## 5. JavaScript 스타일 가이드

JavaScript 코드는 브라우저가 실행하지만, 사람도 함께 읽고 수정한다. 따라서 문법만 맞추는 것이 아니라 일관된 작성 규칙을 지키는 것이 중요하다.

기본적으로 다음 규칙을 지키는 것이 좋다.

1. 변수명과 함수명은 역할이 드러나도록 작성한다.
2. 값이 바뀌지 않는 변수는 `const`, 값이 바뀌는 변수는 `let`을 사용한다.
3. 비교할 때는 가능하면 `==`보다 `===`를 사용한다.
4. 들여쓰기와 줄바꿈을 일정하게 유지한다.
5. HTML은 구조, CSS는 디자인, JavaScript는 동작을 담당하도록 역할을 분리한다.
6. 복잡한 코드에는 필요한 만큼만 주석을 작성하여 의도를 설명한다.
