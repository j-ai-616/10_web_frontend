# 00. 웹 화면 구현 환경 이해

## 1. 화면 구현에서 다루는 세 가지 기술

웹 화면은 보통 HTML, CSS, JavaScript가 함께 동작하여 만들어진다.

HTML은 화면의 구조를 담당한다.
예를 들어 제목, 문단, 목록, 표, 이미지, 입력 양식과 같은 요소를 문서에 배치한다.

CSS는 화면의 디자인을 담당한다.
예를 들어 글자 크기, 색상, 여백, 배경, 배치, 반응형 레이아웃을 지정한다.

JavaScript는 화면의 동작을 담당한다.
예를 들어 버튼 클릭, 입력값 검사, 화면 내용 변경, 목록 추가/삭제 같은 동작을 처리한다.

## 2. 웹 브라우저의 역할

웹 브라우저는 HTML 파일을 읽고 화면으로 보여주는 프로그램이다.
브라우저는 HTML을 해석하면서 필요한 CSS 파일과 JavaScript 파일도 함께 불러온다.

기본 흐름은 다음과 같다.

1. 브라우저가 HTML 파일을 읽는다.
2. HTML 안에 연결된 CSS 파일을 읽고 스타일을 적용한다.
3. HTML 안에 연결된 JavaScript 파일을 읽고 동작을 실행한다.
4. 최종 결과를 화면에 표시한다.

## 3. VS Code에서 HTML 파일 실행하기

수업에서는 VS Code에서 HTML 파일을 작성하고 브라우저로 실행한다.

가장 간단한 방법은 HTML 파일을 브라우저로 직접 여는 것이다.
다만 실습 중에는 Live Server 확장 프로그램을 사용하는 것이 편하다.

Live Server를 사용하면 파일을 수정하고 저장할 때 브라우저 화면을 빠르게 다시 확인할 수 있다.

## 4. 기본 HTML 파일 구조

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>문서 제목</title>
</head>
<body>
    <h1>화면에 보이는 제목</h1>
    <p>화면에 보이는 문단이다.</p>
</body>
</html>
```

`head` 영역은 브라우저와 검색 엔진이 참고하는 정보를 작성하는 영역이다.
`body` 영역은 실제 화면에 보이는 내용을 작성하는 영역이다.

## 5. 개발자 도구 확인하기

브라우저에서 F12 또는 마우스 오른쪽 버튼의 검사 메뉴를 사용하면 개발자 도구를 열 수 있다.

개발자 도구에서는 다음 내용을 확인할 수 있다.

1. Elements 탭: HTML 구조와 CSS 적용 결과 확인
2. Console 탭: JavaScript 출력 및 오류 확인
3. Network 탭: HTML, CSS, JavaScript, 이미지 파일 로딩 확인

수업 초반에는 Console 탭과 Elements 탭을 자주 확인하는 것이 좋다.
